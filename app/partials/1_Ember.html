What is Ember ?

Based on the model-view-controller (MVC) pattern

Designed for Single Page Applications

History

December 2011
The SproutCore 2.0 framework was renamed to Ember.js
August 2015
Ember 2.0 was released

Convention over configuration

A software design paradigm

Why?

Decrease the number of developers' decisions

  • Gaining simplicity
  • Less configuration
  • It is easier to join an existing project
  • It seeks to decrease the number of decisions that developers need to make, gaining simplicity, and not necessarily losing flexibility.
